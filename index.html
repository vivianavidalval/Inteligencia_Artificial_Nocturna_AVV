Computación en la nube:

Es un conjunto de hardware y software, almacenamiento de servicios e interfaces que reciben información como un servicio. Los servicios ofrecidos a clientes, usuarios, empresas, organizaciones y gobierno son plataformas como servicio (Paas), infraestructura como servicio (IaaS), software como servicio (SaaS), adicional los modelos de despliegue como nube privada, nube comunitaria, nube publica y nube híbrida tiene alto impacto en lo económico y social, debido al crecimiento, se aplica desde la digitalización de procesos hasta implementación de soluciones de Cyber seguridad, también su uso esta orientado a incrementar la productividad, reducir los tiempos al impulsar la automatización con el acceso a información confiable y oportuna para la toa de decisiones estratégicas. La sociedad se re configura a medida que la innovación tecnológica avanza articulado a los servicios en la nube. Estos servicios en la nube nos permiten acceder de manera facil a informacion y docuentos en cualquier monento y lugar, atraves de pc, celular uotro medio que se conecte a la nube.

Ciberseguridad:

La ciberceguridad se refiere a explorar las principales vulnerabilidades y con ello crear las mejores estrateias y practicas para garantizar la seguridad de los servicios en la nube, esta seguridad permite proteger los datos y la privacidad de los usuarios. En la nuve hay bastantes riesgos incluidas las vulnerabilidades de seguridad, fugas de datos y accesos no autorizados. Principales amenazas: Acceso no aoutorizado Ataque DdoS (Denegacion de Servicio Distribuidad) fugas de datos Inyeccion de codigo las tendencias y avances en la nube son las tecnologias emergente como IA, blockchain estan ofreciendo nuevas oportunidades de fortalecer la proteccion de datos y la deteccion de amenazas, pero asi como esto beneficia en la seguridad, tambien a los ciberdelincuentes le ayuda para mejorar sus ataques. El desafio hoy endia seria la informatica cuantica y la IA, que requieren de una constante innovacion y estrategias de seguridad que se pueda tener en la nube.

Inteligencia Artificial:

La iteligencia artificial son sistemas informaticos que convierten Datos e informacion en conocimiento. Se debe priorizar los casos de uso de la inteligencia artificial, dependiendo el uso puede haber una mejora en el area de trabajo, atencion, area de investigacion, etc.

La inteligencia artificial (IA) se centra en desarrollar sistemas que imitan la inteligencia humana para realizar tareas como el reconocimiento de patrones, la toma de decisiones y el aprendizaje automático. La IA se divide en dos categorías principales: IA débil, que se especializa en tareas específicas, y IA fuerte, que aspira a tener una comprensión y razonamiento general similar al humano.

La IA está revolucionando diversas industrias al automatizar procesos, mejorar la eficiencia operativa y permitir el análisis avanzado de grandes volúmenes de datos. Aplicaciones comunes incluyen asistentes virtuales, sistemas de recomendación y vehículos autónomos.

Conclusiones:

Podemos decir que: Tanto la nube, la IA y la ciberceguridad van de la mano para que todo funcione de la manera adecuada y se pueda aprovechar de la mejor manera.

Inteligencia Artificial

Resumen

La Inteligencia Artificial (IA) es una rama de la informática que se enfoca en la creación de sistemas capaces de realizar tareas que requieren inteligencia humana, como el reconocimiento de voz, la toma de decisiones y la traducción de idiomas. La IA está transformando industrias enteras, desde la salud hasta las finanzas, al automatizar procesos y mejorar la eficiencia.

¿Qué aprendí en el curso?

En el curso aprendí los conceptos básicos y avanzados de la Inteligencia Artificial, cómo se entrenan los modelos de aprendizaje automático y las diferencias entre los distintos tipos de IA. También comprendí la importancia de los datos en el entrenamiento de modelos y cómo se pueden aplicar las técnicas de IA en la vida real.

Inteligencia Artificial Generativa vs. Redes Neuronales

La Inteligencia Artificial Generativa se refiere a modelos que pueden generar contenido nuevo, como imágenes, texto o música, a partir de patrones aprendidos de datos existentes. Un ejemplo de esto son los modelos de lenguaje como GPT-4, que pueden generar texto coherente y creativo en respuesta a entradas.

Por otro lado, las Redes Neuronales son una técnica específica dentro del campo de la IA inspirada en el funcionamiento del cerebro humano. Estas redes están compuestas por capas de nodos o "neuronas" que procesan información y aprenden a partir de datos. Las Redes Neuronales son la base de muchos avances en la IA, incluidos los sistemas de reconocimiento de voz y visión por computadora.

Red Neuronal para Conversión de Temperaturas

1. Importación de bibliotecas

                import tensorflow as tf
                import numpy as np

TensorFlow (tf): Biblioteca para crear y entrenar modelos de aprendizaje automático.

NumPy (np): Biblioteca para manejar arreglos y operaciones matemáticas.

2. Preparación de datos

                celsius = np.array([-40, -10, 0, 8, 15, 22, 38], dtype=float)
                fahrenheit = np.array([-40, 14, 32, 46, 59, 72, 100], dtype=float)

Creamos dos arreglos con temperaturas en Celsius y sus equivalentes en Fahrenheit.

Estos datos se usarán para entrenar la red neuronal.

3. Definición de la estructura de la red neuronal

                oculta1 = tf.keras.layers.Dense(units=3, input_shape=[1])
                oculta2 = tf.keras.layers.Dense(units=3)
                salida = tf.keras.layers.Dense(units=1)
                modelo = tf.keras.Sequential([oculta1, oculta2, salida])

oculta1: Primera capa oculta con 3 neuronas, acepta una entrada (temperatura en Celsius).

oculta2: Segunda capa oculta con 3 neuronas.

salida: Capa de salida con 1 neurona (temperatura en Fahrenheit).

modelo: Unimos las capas en un modelo secuencial.

4. Compilación del modelo

                modelo.compile(
                optimizer=tf.keras.optimizers.Adam(0.1),
                loss='mean_squared_error'
                )

optimizer: Adam, ajusta los pesos de la red durante el entrenamiento.

loss: Error cuadrático medio, mide la diferencia entre predicciones y valores reales.

5. Entrenamiento del modelo

                print("Comenzando entrenamiento...")
                historial = modelo.fit(celsius, fahrenheit, epochs=1000, verbose=False)
                print("Modelo entrenado!")

Entrenamos el modelo durante 1000 épocas (iteraciones).

historial: Almacena la evolución del error durante el entrenamiento.

6. Visualización del entrenamiento

                import matplotlib.pyplot as plt
                plt.xlabel("# Epoca")
                plt.ylabel("Magnitud de pérdida")
                plt.plot(historial.history["loss"])

Graficamos cómo disminuye el error a lo largo del entrenamiento.

7. Realización de una predicción

                print("Hagamos una predicción!")
                resultado = modelo.predict([100.0])
                print("El resultado es " + str(resultado) + " fahrenheit!")

Usamos el modelo entrenado para predecir la temperatura en Fahrenheit para 100°C.

8. Inspección de los pesos del modelo

                print("Variables internas del modelo")
                print(oculta1.get_weights())
                print(oculta2.get_weights())
                print(salida.get_weights())

Mostramos los pesos aprendidos por cada capa durante el entrenamiento.

Estos pesos determinan cómo la red transforma las entradas en predicciones.

Punto #5: Explicación del Archivo logic.py

Propósito General

Este archivo define una estructura para trabajar con proposiciones lógicas y operadores básicos como AND, OR, NOT, junto con un verificador de modelos.

Contenido del Código

1. Encabezado

import itertools

Importación: Se importa el módulo itertools, que proporciona herramientas para iteradores eficientes.

2. Clase Sentence


    class Sentence():
        """
        Clase base para representar una oración lógica.
        """

Es la clase base que define la estructura de una proposición lógica. Las subclases como Not y And heredan de ella.

Métodos de Sentence


    def evaluate(self, model):
        """
        Evalúa la oración utilizando un modelo.
        """
        raise Exception("nada que evaluar")

Método abstracto sobrescrito por subclases para evaluar las proposiciones en un modelo específico.

3. Clase Symbol


    class Symbol(Sentence):
        """
        Representa un símbolo lógico.
        """
        def __init__(self, name):
            self.name = name
    
        def evaluate(self, model):
            """
            Evalúa el valor del símbolo en un modelo.
            """
            return bool(model.get(self.name, False))

4. Operadores Lógicos

Negación: Clase Not


    class Not(Sentence):
        """
        Representa una negación lógica.
        """
        def __init__(self, operand):
            Sentence.validate(operand)
            self.operand = operand
    
        def evaluate(self, model):
            return not self.operand.evaluate(model)

Conjunción: Clase And


    class And(Sentence):
        """
        Representa una conjunción lógica.
        """
        def __init__(self, *conjuncts):
            self.conjuncts = list(conjuncts)
    
        def evaluate(self, model):
            return all(conjunct.evaluate(model) for conjunct in self.conjuncts)

Disyunción: Clase Or


    class Or(Sentence):
        """
        Representa una disyunción lógica.
        """
        def __init__(self, *disjuncts):
            self.disjuncts = list(disjuncts)
    
        def evaluate(self, model):
            return any(disjunct.evaluate(model) for disjunct in self.disjuncts)

5. Verificación de Modelos: model_check


    def model_check(knowledge, query):
        """
        Verifica si una base de conocimiento implica una consulta.
        """
        def check_all(knowledge, query, symbols, model):
            if not symbols:
                if knowledge.evaluate(model):
                    return query.evaluate(model)
                return True
            else:
                p = symbols.pop()
                model_true, model_false = model.copy(), model.copy()
                model_true[p], model_false[p] = True, False
                return (check_all(knowledge, query, symbols, model_true) and
                        check_all(knowledge, query, symbols, model_false))
    
        symbols = set.union(knowledge.symbols(), query.symbols())
        return check_all(knowledge, query, symbols, dict())
